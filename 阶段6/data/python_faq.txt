Python 常见问题解答（FAQ）

第一章：基础语法

Q: Python 中列表和元组有什么区别？
A: 列表（list）是可变的，用方括号 [] 定义；元组（tuple）是不可变的，用圆括号 () 定义。列表适合需要增删改的场景，元组适合作为字典键或不需要修改的数据。

Q: Python 中的深拷贝和浅拷贝有什么区别？
A: 浅拷贝（copy.copy）只复制对象本身，内部的可变对象仍指向原引用；深拷贝（copy.deepcopy）递归复制所有层级，完全独立于原对象。

Q: Python 中 *args 和 **kwargs 的区别？
A: *args 将多余的位置参数收集为元组，**kwargs 将多余的关键字参数收集为字典。例如 def f(*args, **kwargs) 可以接受任意参数。

Q: Python 中的装饰器是什么？
A: 装饰器是一种设计模式，用 @decorator 语法糖将一个函数作为参数传入另一个函数，在不修改原函数代码的情况下增强其功能。常见用途：日志、权限检查、缓存。

第二章：数据结构

Q: Python 字典的底层实现是什么？
A: Python 3.7+ 的字典使用哈希表实现，保持插入顺序。键通过哈希函数映射到数组下标，查找时间复杂度平均为 O(1)。

Q: collections 模块有哪些常用数据结构？
A: defaultdict（带默认值的字典）、Counter（计数器）、OrderedDict（有序字典）、deque（双端队列）、namedtuple（命名元组）。

Q: 如何对列表去重并保持顺序？
A: 使用 dict.fromkeys(list) 或 {x: None for x in list}（Python 3.7+ 字典保序），或使用列表推导 + seen 集合：seen = set(); [x for x in lst if not (x in seen or seen.add(x))]。

第三章：函数与面向对象

Q: Python 中类方法和静态方法的区别？
A: 类方法（@classmethod）第一个参数是 cls，可以访问和修改类状态；静态方法（@staticmethod）没有隐式参数，相当于普通函数放在类命名空间里。

Q: 什么是 Python 的魔术方法？
A: 魔术方法（dunder methods）是以双下划线开头和结尾的特殊方法，如 __init__（初始化）、__str__（字符串表示）、__len__（长度）、__getitem__（索引访问）等，用于实现运算符重载和协议。

Q: Python 的多重继承如何处理冲突？
A: Python 使用 C3 线性化算法（MRO）确定方法解析顺序，可通过 ClassName.__mro__ 查看。子类方法优先，同层级按声明顺序从左到右。

第四章：异步与并发

Q: async/await 的工作原理是什么？
A: async def 定义协程函数，调用返回协程对象；await 挂起当前协程等待结果，事件循环在协程间切换实现并发。本质是协作式多任务。

Q: GIL 是什么？对多线程有什么影响？
A: GIL（全局解释器锁）是 CPython 的互斥锁，同一时刻只有一个线程执行 Python 字节码。CPU 密集型任务无法通过多线程加速，但 I/O 密集型任务在等待时可以释放 GIL，多线程仍然有效。

Q: asyncio.gather 和 asyncio.wait 的区别？
A: gather 按顺序返回结果列表，wait 返回 (done, pending) 集合。gather 更简单直观，wait 更灵活，支持 FIRST_COMPLETED 等策略。

第五章：Web 开发

Q: Flask 和 Django 的主要区别？
A: Flask 是微框架，轻量灵活，按需选择组件；Django 是全栈框架，内置 ORM、Admin、Auth 等，适合快速开发。小型项目选 Flask，大型项目选 Django。

Q: 什么是 WSGI 和 ASGI？
A: WSGI（Web Server Gateway Interface）是 Python Web 应用的同步接口标准；ASGI（Asynchronous Server Gateway Interface）是异步版本，支持 WebSocket 等长连接。Django 3.0+ 支持 ASGI。

Q: 如何处理跨域请求（CORS）？
A: 使用 flask-cors 或 django-cors-headers 中间件，设置 Access-Control-Allow-Origin 等响应头。开发环境可允许所有来源，生产环境应限制白名单。

第六章：测试与调试

Q: pytest 和 unittest 的区别？
A: unittest 是 Python 内置的 xUnit 风格测试框架，需继承 TestCase；pytest 更简洁，用 assert 语句，支持 fixture、参数化、插件生态更丰富。

Q: 如何调试 Python 程序？
A: 方法包括：1) print/logging 语句；2) pdb 断点调试（import pdb; pdb.set_trace()）；3) IDE 集成调试器；4) cProfile 性能分析；5) memory_profiler 内存分析。

Q: 什么是类型提示（Type Hints）？
A: Python 3.5+ 支持用类型注解标注函数参数和返回值类型，如 def f(x: int) -> str。类型提示不影响运行时行为，但可被 mypy 等工具静态检查，提高代码可读性和安全性。
