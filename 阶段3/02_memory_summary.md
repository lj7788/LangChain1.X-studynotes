# 02_memory_summary.py

## 功能说明

`ConversationSummaryMemory` 使用 LLM 生成对话摘要，适合长对话场景，可以有效减少记忆占用的 token 数量。

## 核心特性

- **自动摘要**: 使用 LLM 将对话历史压缩成摘要
- **节省 Token**: 比完整保存对话更节省上下文空间
- **保留关键信息**: 摘要保留重要的上下文信息

## 关键参数

| 参数 | 说明 |
|------|------|
| llm | 用于生成摘要的 LLM |
| memory_key | 在 prompt 中引用记忆的变量名 |
| return_messages | True 返回消息列表，False 返回字符串 |

## ConversationBufferMemory vs ConversationSummaryMemory

| 特性 | ConversationBufferMemory | ConversationSummaryMemory |
|------|---------------------------|----------------------------|
| 存储方式 | 完整对话 | 摘要 |
| Token 消耗 | 较高 | 较低 |
| 信息完整度 | 100% | 取决于摘要质量 |
| 适用场景 | 短对话 | 长对话 |

## 运行示例

```bash
python 阶段3/02_memory_summary.py
```

## 输出示例

```
=== 对话 1 ===
用户: 我叫张三，是一名软件工程师。我喜欢编程和读书。
助手: 你好张三！很高兴认识你。作为一名软件工程师，编程和读书都是很好的爱好...

=== 对话 2 ===
用户: 你喜欢什么运动？我喜欢打篮球。
助手: 抱歉，我作为AI没有真正的喜好。不过你提到喜欢打篮球，这是一项很棒的运动...

=== 查看摘要记忆 ===
记忆内容:
人类与AI进行了一段对话。人类介绍自己是张三，是一名软件工程师，喜欢编程、读书和打篮球...

=== 对话 3 ===
用户: 总结一下我告诉你的关于我自己的信息
助手: 你告诉我以下关于你自己的信息：
1. 名字叫张三
2. 职业是软件工程师
3. 喜欢编程
4. 喜欢读书
5. 喜欢打篮球
```
